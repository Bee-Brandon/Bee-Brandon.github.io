Case Study: CalJOBS Automation

Reducing documentation time by 80% at East LA AJCC

Overview

Business Service Representatives at California's America's Job Centers spend hours each week on CalJOBS documentation - the state's workforce development case management system. I built two tools that automate the most time-consuming parts of this work.

The Problem

Staff were losing significant time to two repetitive workflows:

1. Activity Code Lookup

CalJOBS has 326+ activity codes. Every client interaction needs to be logged with the correct code. Staff would scroll through lists, consult printed guides, or ask colleagues - often taking 2-5 minutes just to find the right code.

2. Activity Creation Workflow

Creating a single activity record required 30+ clicks across 7 different screens:

  1. Login and navigation to client search
  2. Search for participant by State ID
  3. Navigate through Staff Profiles → Case Management → Programs
  4. Find the correct WIOA program entry
  5. Open Activities section
  6. Fill form fields (Program Group, Office, Agency, Grant, Activity Code, Dates)
  7. Add case notes with Subject Line and Description

Time cost per activity: 5-8 minutes of clicking, waiting, and typing the same information repeatedly.

The Solution

Tool 1: CalJOBS Code Assistant

A web-based AI assistant that eliminates code lookup time:

  • AI Code Suggestions - Describe what you did in plain English, get the correct code
  • Case Note Generator - Select a code, instantly get formatted Subject Line + Description
  • Quick Actions - One-click access to most common codes
  • Fuzzy Search - Type partial words, find matching codes instantly

Example: Staff types "helped client with resume" → AI suggests Code 115 (Resume Preparation) → Click to generate case note ready to paste.

Tool 2: CalJOBS Automation Bot

A 7-phase automation that handles the entire activity creation workflow:

Phase What It Automates
1. Login Authenticates to CalJOBS securely
2. Search Enters State ID, finds participant
3. Navigate Clicks through to Case Management → Programs
4. Find WIOA Locates the correct program entry
5. Activities Opens the Activities section
6. Fill Form Selects all dropdowns, enters dates
7. Finalize Adds case note, saves, confirms

Key features: Real-time progress display, review page before submission, demo mode for safe testing, error handling with screenshots.

Technical Implementation

Python Streamlit Playwright Claude AI API Fuzzy Matching Windows Credential Manager

Architecture Decisions

  • Web-based interface - No installation required, staff access via browser
  • Local deployment - Runs on office network, no client data leaves premises
  • AI with embedded context - All 326 codes in prompt for accurate suggestions
  • Secure credentials - Passwords stored in Windows Credential Manager, never plaintext
  • Demo mode by default - Bot won't touch real data until explicitly enabled

Results

Before

  • 2-5 min to find correct code
  • 2-3 min to write case note
  • 30+ clicks per activity
  • 5-8 min per activity total
  • Frequent wrong code selections

After

  • 10 seconds to find code
  • 10 seconds for case note
  • 0 clicks (automated)
  • ~40 seconds per activity
  • AI suggests correct codes
